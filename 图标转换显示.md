### 2.1 svg\_to\_png.py - SVG转PNG再转RGB565

**功能**: 将和风天气SVG图标转换为PNG，再转为RGB565格式的C数组头文件

**使用方法**:

```bash
# 默认参数运行（自动查找QWeather-Icons-1.8.0/icons目录）
python svg_to_png.py

# 指定SVG目录和输出目录
python svg_to_png.py --svg-dir ./icons --png-dir ./weather_png --header-dir ./weather_icons

# 指定图标尺寸和要转换的图标代码
python svg_to_png.py --size 64 --codes 100 101 102 800
```

**参数说明**:

- `--svg-dir`: SVG图标目录（默认自动查找）
- `--png-dir`: PNG输出目录（默认: weather\_png）
- `--header-dir`: C头文件输出目录（默认: weather\_icons）
- `--size`: 图标尺寸（默认: 64）
- `--codes`: 指定要转换的图标代码列表
- `--skip-png`: 跳过PNG生成步骤

**输出格式**: RGB565 (16-bit)，每个像素2字节，存储在 PROGMEM 中

***

### 2.2 merge\_weather\_icons.py - 合并图标头文件

**功能**: 将 `weather_icons` 目录下的多个单独头文件合并为一个完整的头文件

**使用方法**:

```bash
# 默认参数运行
python merge_weather_icons.py

# 指定输入目录和输出文件
python merge_weather_icons.py --input ./weather_icons --output ./include/weather_icons.h
```

**输出格式**: RGB565 (16-bit)，包含所有图标数组和 `getWeatherIcon()` 函数

***

## 三、完整转换流程

```bash
# 1. SVG转PNG，同时生成单独的RGB565头文件
python svg_to_png.py --size 64

# 2. 合并所有头文件为一个
python merge_weather_icons.py
```

**输出文件**:

- `weather_png/icon_XXX.png` - 中间PNG文件
- `weather_icons/weather_icon_XXX.h` - 单独图标头文件
- `include/weather_icons.h` - 合并后的完整头文件

## 三、在 Arduino/ESP32 中使用

### 3.1 图标显示

```cpp
#include <TFT_eSPI.h>
#include "weather_icons.h"

TFT_eSPI tft = TFT_eSPI();

void setup() {
    tft.begin();
    tft.fillScreen(TFT_BLACK);
    
    // 根据天气代码获取图标数组
    const uint16_t* icon = getWeatherIcon(100);  // 100=晴
    
    // 显示图标（x, y, 宽度, 高度, 数据数组）
    tft.pushImage(0, 0, WEATHER_ICON_WIDTH, WEATHER_ICON_HEIGHT, icon);
}
```

## 四、图标代码说明

和风天气图标代码对应：

| 代码范围    | 天气类型         |
| ------- | ------------ |
| 100-104 | 晴、多云         |
| 150-153 | 雾、霾          |
| 300-399 | 小雨、中雨、大雨、暴雨  |
| 400-499 | 雨夹雪、小雪、中雪、大雪 |
| 500-515 | 霜、雾凇、冰雹      |
| 800-807 | 晴、少云、多云、阴    |
| 900-999 | 特殊天气         |

## 五、在线转换工具

如果仅需转换单个文件，可使用在线工具：

**Notisrac's File to C Array Converter**:

- 地址: <https://notisrac.github.io/FileToCArray/>
- 使用方法:
  1. 上传图片
  2. **Output type** → 选择 `uint16_t`
  3. **Byte order** → 选择 `Little Endian`
  4. 格式: `16bit RRRRRGGGGGGBBBBB` (RGB565)
